7 Spark Sql

John Xu
Spark系列二

发布于：2020年10月14日
次浏览

使用Spark SQL，我们可以对存储到数据库中的视图或表进行SQL查询，还可以使用系统函数或用户定义函数来分析查询计划以优化其工作负载。

SQL（结构化查询语言，Structured Query Language）是一种表示数据关系操作的特定领域语言，其广泛应用在关系型数据库中。

许多“NoSQL”数据库也支持类SQL语言。

在Spark流行之前，Apache Hive 是支持SQL的主流大数据处理工具，但现在大量用户都在使用Spark SQL。

2. Catalog

Catalog 是Spark SQL中最高级别的抽象，用于存储用户数据中的元数据以及其他有用的东西，如数据库，数据表，函数和视图。

Catalog 在 org.apache.spark.SQL.catalog.Catalog包中，它包含很多的函数，用于执行诸如列举表、数据库和函数之类的操作。

2.1. 数据表

数据表在逻辑上等同于DataFrame，它们的核心区别在于，DataFrame是编程语言范围内定义的，而数据表是在数据库中定义的。

在 Spark2.X 中，数据表始终是实际包含数据的，没有类似视图的临时表概念，只有视图不包含数据，因此如果你要删除一个表，那么可能会丢失数据，

Spark托管表

表存储两类重要的信息，表中的数据和元数据。Spark既可以管理一组的元数据，也可以管理实际数据。

当定义磁盘上的若干文件作为一个数据表时，这个就是非托管表（unmanaged table）；在DataFrame上使用saveAsTable来创建一个数据表时，就是创建了一个托管表（managed table），spark将跟踪托管表的所有相关信息。

2.2. 视图

定义视图即基于现有表的一组转换操作，基本上只是保存查询计划，这可以方便地组织或重用查询逻辑。

Spark中，视图可以是全局的，也可以是针对某个数据库的或针对每个会话的。

对于终端用户，视图可以被视为表，但是数据并没有重写到新位置，它们只是在查询时对数据源执行转换操作（filter, select, 或者更大规模的GROUP BY 或 ROLLUP）:

3.3. 子查询

Spark中有两个基本子查询：

  • 相关子查询（Correlated Subquery）， 使用来自查询外的一些信息
  • 不相关子查询（Uncorrelated Subquery）， 不包括外部的信息

每个子查询可以返回单个值或多个值。

Spark包括对谓词子查询（Predicate Subquery）的支持，它允许基于值进行筛选。
