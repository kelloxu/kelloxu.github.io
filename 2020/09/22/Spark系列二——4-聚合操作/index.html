Spark系列二——4-聚合操作

John Xu
Spark系列二

发布于：2020年9月22日
次浏览

一般情况下，用户使用聚合操作对数据分组后的各组内的数值型数据进行汇总，这个汇总运算可能是求和、累乘、或者是简单的计数。

Spark 可以将任何类型的值聚合成为 array 数组、list列表或 map 映射。

除了处理任意类型的值之外，Spark 还允许创建以下分组类型：

  • group by

指定一个或多个key，也可以指定一个或多个聚合函数，来对包含 value 的列执行转换操作。

  • window

指定一个或多个key，也可以指定一个或多个聚合函数，来对包含 value 的列执行转换操作，但是输入到当前函数的行与当前行有某种联系。

  • gpoup setting

用于在多个不同场景进行聚合。grouping set 是 SQL 中的一个保留字，而在 DataFrame 中需要使用 rollback 和 cube。

  • rollup

指定一个或多个key，也可以指定一个或多个聚合函数，来对包含 value 的列执行转换操作，并针对指定的多个 key 进行分级分组汇总。

  • cube

指定一个或多个key，也可以指定一个或多个聚合函数，来对包含 value 的列执行转换操作，并针对指定的多个 key 进行全组合分组汇总。

数据准备：

可以使用 count 获得数据集的总体大小，因为它是动作操作，所以会立即执行，也因此它还有一个作用，可以缓存整个 DataFrame 到内存里（前提是先执行了 df.cache 或 df.persist）。

这里的 count 与 functions 包里的 count() 函数不一样，functions 包里的 count() 函数是转换操作。

1. 聚合函数

除了 DataFrame 的某些操作或通过 .stat 访问的方法（如Spark系列二——3-处理不同的数据类型介绍的），所有聚合操作都是以函数形式出现。

大多数聚合函数可以在 org.apache.spark.sql.functions 包中找到。

count

由上图可知，functions.count 是转换操作。

functions 包里的 count() 函数是转换操作。

countDistinct

仅在统计针对某列的计数时才有意义。

approx_count_distinct

通常在大致估计一个数据集的时候，不需要精确统计，某种精度的近似值是可以接受的。

approx_count_distinct 带了另外一个参数，指定可容忍的最大误差， 本例中 0.1 指定了一个相当大的误差率，因此得到的答案和正确值差距很大，但执行速度很快，比 countDistinct 函数执行耗时更少。

误差率越大，执行速度越快；误差率越小，执行速度越慢。

first、last

这两个函数可以得到 DataFrame 的第一个值和最后一个值，它是基于 DataFrame 中行的顺序，而不是值的顺序。

min、max

取某一列的最小值和最大值。

sum

累加某一列所有行的值。

sumDistinct

去重后求和。

avg、mean

获取平均值。

方差和协方差

方差：各数据样本与均值之间差的平方的平均值。
标准差：方差的平方根。

Spark 既支持统计样本标准差，也支持统计总体标准差，它们在统计学上是完全不同的概念，一定要区分它们。

  • var_pop 总体方差
  • var_samp 样本方差
  • stddev_pop 总体标准差
  • stddev_samp 样本标准差

skewness、kurtosis

偏度系数（skewness）和峰度系数（kurtosis）都是对数据集中极端数据点的衡量指标，在将数据建模为随机变量的概率分布时，它们都很重要。

偏度系数：衡量相对于平均值的不对称程度；峰度系数：衡量数据分布形态陡缓程度。

协方差和相关性

我们讨论了单列聚合，不过有的函数是比较两个不同列的值之间的相互关系，cov（协方差） 和 corr（相关性） 就是其中两种。

相关性采用 Person 相关系数来衡量，范围是 -1 ～ 1；协方差的范围由数据中的输入决定。

跟 var 函数一样，协方差又分为样本协方差和总体协方差，但是相关性没有这个概念。

聚合输出复杂类型

在 Spark 中，不仅可以在数值型上执行聚合操作，还能在复杂类型上执行聚合操作，例如，可以收集某列上的值到一个 list 表里，或者将 unique 唯一值收集到一个 set 集合里。用户可以在流水线处理的后续操作中再访问该集合，或者将整个集合传递给用户自定义函数。

3. 用户自定义聚合函数

可以使用 UDAF 来计算输入数据组（与单行相对）的自定义计算。

Spark 维护单个 AggregationBuffer，它用于存储每组输入数据的中间结果。

若要创建 UDAF，必须即成 UserDefinedAggregationFunction 基类并实现以下方法:

  • inputSchema

用于指定输入参数，输入类型为 StructType。

  • bufferSchema

用于指定 UDAF 中间结果，类型为 StrcutType。

  • dataType

用于指定返回结果，类型为 DataType。

  • deterministic

布尔值，它指定此 UDAF 对于某个输入是否会返回相同的结果。

  • initialize

初始化聚合缓冲区的初始值。

  • update

描述应如何根据给定行更新内部缓冲器。

  • merge

描述应如何合并两个聚合缓冲区。

  • evaluate

将生成聚合最终结果。
